## 整体架构

```
请求进来
    ↓
SecurityFilterChain（所有请求先过这里）
    ↓ 有 Bearer Token → JwtDecoder 验签
    ↓ 有 JSESSIONID  → 读 Session（OIDC 或 OA）
    ↓
URL 授权检查
    ↓
Controller（路由处理）
    ↓
Service（业务逻辑）
    ↓
Repository（数据库）
```

项目同时维护三套独立身份：

| 身份 | 登录入口 | 存储方式 | 用途 |
|---|---|---|---|
| OA Session | `POST /api/oa/login` | JSESSIONID Cookie | OA 门户 |
| Keycloak OIDC | `/oauth2/authorization/keycloak` | JSESSIONID Cookie | SSO 跨系统 |
| JWT Bearer | `POST /api/auth/login` | Authorization Header | API 调用 |

## 第一层：入口文件

**文件**：`IamApplication.java`

```java
@SpringBootApplication   // = @Configuration + @EnableAutoConfiguration + @ComponentScan
```

Bean = 由 Spring 管理生命周期的对象。你不用 `new`，Spring 负责创建、注入、销毁。

```js

```

## 第三层：SecurityConfig — 安全中枢

**文件**：`config/SecurityConfig.java`

JS 类比：Express 中间件链

```java
@Bean
public SecurityFilterChain securityFilterChain(HttpSecurity http) {
    http
        // 1. URL 授权规则
        .authorizeHttpRequests(auth -> auth
            .requestMatchers("/oa.html", ...).permitAll()       // 公开
            .requestMatchers("/api/admin/**").hasRole("ADMIN")  // 需要 ADMIN
            .anyRequest().authenticated()                        // 其余要登录
        )
        // 2. JWT 验证（API 认证）
        .oauth2ResourceServer(oauth -> oauth.jwt(...))
        // 3. Keycloak SSO（按需开启）
        if (clientRegistrationRepositoryProvider.getIfAvailable() != null) {
            http.oauth2Login(oauth -> oauth.defaultSuccessUrl("/sso/launch"));
        }
}
```

**注意点**：

- `@Bean` = 把方法返回值注册为单例，其他地方需要时 Spring 自动注入
- `ObjectProvider.getIfAvailable()` = 懒加载，Bean 不存在时返回 null 而非报错
- Keycloak 未配置时 `oauth2Login` 不注册，`/oauth2/authorization/keycloak` 不存在

**常用注解对照**：

| Spring 注解 | Express 等价 |
|---|---|

## 第五层：JwtService — Token 工厂

**文件**：`security/JwtService.java`

```java
@Service  // 业务逻辑层
public class JwtService {

    public TokenResponse issueToken(User user) {
        JwtClaimsSet claims = JwtClaimsSet.builder()
            .issuer("iam-starter")   // iss
            .subject(user.getUsername())  // sub
            .expiresAt(now.plus(60, MINUTES))  // exp
            .claim("roles", roles)   // 自定义 claim
            .build();

        String token = jwtEncoder.encode(...).getTokenValue();
        return new TokenResponse(token, "Bearer", 3600, ...);
    }
}
```

**Builder Pattern**（Java 没有对象字面量，用链式构造代替）：

```java
// Java
JwtClaimsSet claims = JwtClaimsSet.builder()
    .subject("zhangsan")
    .build();

// JS 等价
const claims = { sub: 'zhangsan' }
```

**`@Service` / `@Controller` / `@Repository` 的区别**：功能完全一样（都是 Bean），只是语义分层，帮助读代码时快速定位职责。

## 第六层：Entity + Repository — 数据库层

**文件**：`entity/User.java`、`repository/UserRepository.java`

### Entity = 表结构 + 类型定义

```java
@Entity
@Table(name = "users")
@Data           // 自动生成 getter/setter
@Builder        // 支持 User.builder().username("x").build()
public class User {

    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;  // 自增主键

    @Column(nullable = false, unique = true, length = 64)
    private String username;  // NOT NULL UNIQUE VARCHAR(64)

    @PrePersist  // INSERT 前自动触发
    void onCreate() { this.createdAt = Instant.now(); }
}
```

Drizzle 对照：
```ts
export const users = pgTable('users', {
  id: serial('id').primaryKey(),
  username: varchar('username', { length: 64 }).notNull().unique(),
})
```

### Repository = 零 SQL 的数据库访问

```java
// 只写接口，不写实现
public interface UserRepository extends JpaRepository<User, Long> {
    Optional<User> findByUsername(String username);
    // Spring 自动生成：SELECT * FROM users WHERE username = ?
}
```

方法名命名规则：

| 方法名 | 生成的 SQL |
|---|---|
| `findByUsername` | `WHERE username = ?` |
| `findByEmailAndStatus` | `WHERE email = ? AND status = ?` |
| `findByStatusOrderByCreatedAtDesc` | `WHERE status = ? ORDER BY created_at DESC` |
| `existsByEmail` | `SELECT count(*) > 0 WHERE email = ?` |

`JpaRepository` 免费附赠：`save()`、`findById()`、`findAll()`、`deleteById()`、`count()` 等。

**`Optional<User>`**：Java 强制处理"可能为空"的情况：

```java
userRepository.findByUsername("x")
    .orElseThrow(() -> new ResponseStatusException(401, "..."))  // 找不到抛异常
    .orElse(defaultUser)   // 找不到用默认值
```

## 横向：OaController — 独立表单登录

**文件**：`controller/OaController.java`

OA 系统有自己的用户表（`oa_users`），不走 Spring Security，完全手动管理 Session。

```java
@PostMapping("/login")
public ResponseEntity<?> login(@RequestBody LoginRequest req, HttpSession session) {
    // 1. 查库验密码
    OaUser user = oaUserRepository.findByUsername(req.username()).orElseThrow();
    if (!passwordEncoder.matches(req.password(), user.getPasswordHash())) {
        return ResponseEntity.status(401)...;
    }
    // 2. 写 Session（相当于 req.session.OA_USER = user）
    session.setAttribute("OA_USER", OaUserDto.from(user));
    return ResponseEntity.ok(dto);
}
```

**为什么用 DTO 而不直接返回 Entity**：

```
OaUser（含 passwordHash）→ OaUserDto（只含安全字段）→ JSON 响应
```

永远不要把含密码的 Entity 直接序列化返回。

**两套 Session 完全独立**：

| | OA Session | OIDC Session |
|---|---|---|
| 登出接口 | `DELETE session["OA_USER"]` | `POST /logout`（Spring Security） |
| 互相影响 | 不影响 | 不影响 |

## 横向：SsoController + TargetController — SSO 流程

**文件**：`controller/SsoController.java`、`controller/TargetController.java`、`sso/TokenRefreshService.java`

### 持有凭证流程

```
OA 门户点「IAM 系统」
    ↓
GET /sso/launch（SsoController）
    ├─ Keycloak 未配置 → /dashboard.html
    ├─ 未登录 → /oauth2/authorization/keycloak → Keycloak 登录 → 回到 /sso/launch
    └─ 已登录
         ├─ token 有效 ──────────────────────────────────────────┐
         ├─ token 过期 → TokenRefreshService.refresh() → 新 token ┤
         └─ refresh 失败 → 重新 SSO                               ↓
                                                GET /target/enter?token=xxx
                                                （TargetController）
                                                    ↓ 调 Keycloak GET /userinfo
                                                    ↓ 验证通过，存 session
                                                    ↓
                                                target.html
```

### SsoController 关键点

```java
// @Autowired(required = false)：Bean 不存在时为 null，不报错
@Autowired(required = false)
private OAuth2AuthorizedClientService authorizedClientService;

// OAuth2AuthorizedClientService：Spring 存 token 的仓库
OAuth2AuthorizedClient client =
    authorizedClientService.loadAuthorizedClient("keycloak", principal.getName());

// 取出 access_token，检查是否过期
if (isExpired(client.getAccessToken())) {
    accessToken = tokenRefreshService.refresh(client);  // POST refresh_token → /token
}

// 携带 token 跳目标系统
response.sendRedirect("/target/enter?token=" + accessToken);
```

### TargetController 关键点

```java
// 用 access_token 调 Keycloak /userinfo 验证身份
private Map<String, Object> callUserInfo(String token) {
    String userinfoUri = reg.getProviderDetails().getUserInfoEndpoint().getUri();

    HttpHeaders headers = new HttpHeaders();
    headers.setBearerAuth(token);  // Authorization: Bearer xxx

    // RestTemplate = Java 的 fetch
    ResponseEntity<Map<String, Object>> res = new RestTemplate().exchange(
        userinfoUri, HttpMethod.GET, new HttpEntity<>(headers),
        new ParameterizedTypeReference<>() {}
    );
    return res.getBody();  // Keycloak 返回的用户信息
}
```

**`@Controller` vs `@RestController`**：

```java
@Controller  // 需要同时有重定向和 JSON 返回时用这个
public class TargetController {

    @GetMapping("/target/enter")
    public void enter(...) {
        response.sendRedirect("/target.html");  // 重定向，不序列化
    }

    @GetMapping("/api/target/me")
    @ResponseBody  // 单独这个方法返回 JSON
    public ResponseEntity<?> me(...) { ... }
}
```

## 纵向：OrgSync — 定时任务 + 组织同步

**文件**：`sync/OrgSource.java`、`sync/OrgSyncService.java`、`sync/OrgSyncScheduler.java`、`sync/MockOrgSource.java`

### 分层结构

```
OrgSyncScheduler    定时触发（每 5 分钟）
    ↓
OrgSyncService      核心逻辑（遍历所有数据源，upsert 到数据库）
    ↓
OrgSource           接口（规范数据源的行为）
    ↑
MockOrgSource       假数据（开发用）
LdapOrgSource       真实 LDAP（自己扩展）
```

### 面向接口编程

```java
// 接口：只定规范
public interface OrgSource {
    String getSourceName();
    List<ExternalUser> fetchUsers();
}

// 任意实现：implements OrgSource
@Component
public class MockOrgSource implements OrgSource { ... }

@Component
public class LdapOrgSource implements OrgSource { ... }  // 你来写
```

### List 注入魔法

```java
// 声明需要所有 OrgSource 实现的列表
// Spring 自动把容器里所有 OrgSource Bean 收集进来
private final List<OrgSource> orgSources;
```

新增一个 `LdapOrgSource`，`OrgSyncService` 代码一行不改，自动同步两个数据源。

### 事务

```java
@Transactional  // 所有数据库操作在一个事务，失败自动回滚
public SyncResult syncAll() { ... }
```

JS 类比：
```js
await db.transaction(async (tx) => { ... })
```

### 定时任务

```java
// Spring cron：6 位（秒 分 时 日 月 周）
@Scheduled(cron = "${ldap.sync-cron:0 */5 * * * *}")
public void scheduledSync() {
    orgSyncService.syncAll();
}
```

### Upsert 模式

```java
// 按 externalId 判断是否已存在，存在则更新，不存在则插入
Optional<User> existing = userRepository.findByExternalId(ext.getExternalId());
if (existing.isPresent()) {
    // update
} else {
    // insert
}
```

用 `externalId` 而不是 `email` / `username`，因为用户可能改名换邮箱，但外部系统 ID 不变。

## 核心概念速查

| Java / Spring | JS 等价 | 说明 |
|---|---|---|
| `@Bean` | `export const x = new X()` | 注册单例到容器 |
| `@Autowired` / 构造器注入 | `import { x }` | 从容器取出 Bean |
| `@Value("${x}")` | `process.env.X` | 读配置 |
| `@RestController` | Express Router | HTTP 入口，自动序列化 JSON |
| `@Service` | `services/*.js` | 业务逻辑层 |
| `@Repository` | `models/*.js` | 数据访问层 |
| `@Entity` | Drizzle/Prisma schema | 表结构定义 |
| `Optional<T>` | `T \| null`（强制处理） | 防 NPE |
| `record` | `interface`（只读数据） | 纯数据结构 |
| `@Transactional` | `db.transaction(...)` | 数据库事务 |
| `@Scheduled` | `cron.schedule(...)` | 定时任务 |
| `RestTemplate` | `fetch` | 发 HTTP 请求 |
| Builder Pattern | 对象字面量 `{ a: 1, b: 2 }` | 链式构造复杂对象 |
| `List<OrgSource>` 注入 | 手动维护数组 | 自动收集所有实现 |
